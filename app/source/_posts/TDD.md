- 传统开发 primitive

  - 流程不清晰
  - 反复确认
  - 人力沟通成本大
  - 代码难以维护

  

- 测试驱动开发 test-driven development(TDD)

  - 先确定需求
  - 再确定验收标准
  - 编写test 
  - 编写业务代码
  - run test
  - submit code

> 这里需要说明一下， 在当下2021年，TDD模式被coder们更偏向为开发完之后再开发 e2e,  unit test来测试自己的代码运行。
